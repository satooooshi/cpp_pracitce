测试点在 traces 文件夹下，cpp 文件为对 List 类的调用，out 文件为该测试点对应的正确输出结果。

因为使用了 Valgrind 作为内存管理的检测工具，而 Valgrind 没有 Windows 版，
所以测试脚本在 Linux / Mac 系统下才能测试内存，在 Windows 系统下只能测试输出内容的正确性。

Linux / Mac 系统下用法：
1. 安装 Valgrind（方法自行查找，一条命令就可以完成）
2. 将你提交的代码放在 submits 文件夹下
3. 在终端中执行 python AutoGrader.py
4. 在 report 文件夹查看测评报告（包括输出正确性和内存情况的信息），如有错误会生成错误详情报告
5. 若修改代码后要评测第二次，请先执行 clean.sh 清理原来生成的文件，否则评测可能失败

Windows 系统下用法：
1. 安装 g++（可下载 MinGW），并把 g++ 所在目录添加至环境变量
2. 将你提交的代码放在 submits 文件夹下
3. 运行 AutoGrader.py（Python 大家肯定都装了）
4. 在 report 文件夹查看测评报告（仅包含输出正确性的信息），如有错误会生成错误详情报告
5. 若修改代码后要评测第二次，请先执行 clean.bat 清理原来生成的文件，否则评测可能失败
6. 要测内存可以安装 Linux 虚拟机（反正你们以后也会用到的），或者自行查找 Windows 平台下可用的内存泄漏检测方法
7. 如果安装 g++ 失败，也可以选择用 VS 手动编译比较 10 个测试点的输出结果是否正确
